WordPress users can be staff (editors, admins) as well as customers. For customer-specific tools see [Customers](/docs/tools-reference/customers).

## store_mcp_list_users

List users with filters. **Tier:** Free.

### Parameters

| Parameter | Type    | Required | Default | Description |
|-----------|---------|----------|---------|-------------|
| search    | string  | no       | —       | Email, name, login |
| role      | string  | no       | —       | WordPress role |
| orderby   | string  | no       | registered | `id`, `name`, `registered` |
| order     | string  | no       | desc    | `asc`, `desc` |
| page      | integer | no       | 1       | Page number |
| per_page  | integer | no       | 10      | Items per page |

## store_mcp_get_user

Get a user by ID, email, or login. **Tier:** Free.

## store_mcp_create_user

Create a user. **Tier:** Free.

## store_mcp_update_user

Update a user (name, email, role, password). **Tier:** Free.

## store_mcp_delete_user

Delete a user. Option to reassign content. **Tier:** Free.

<Callout type="warning" title="Role changes are powerful">
  Promoting a user to `administrator` via AI is convenient but risky. Consider
  keeping role changes behind a manual admin flow.
</Callout>
